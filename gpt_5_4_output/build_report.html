Assumptions

  • OPENAI_API_KEY is available in the local environment for autonomous build and automated UI execution.
  • The local machine has a functioning OS keyring, which the app uses for persistent secure storage of the API key.
  • The target workbook is a single-sheet Excel file compatible with pandas.read_excel.
  • Acceptance validation is intended to run each sample query in isolation; the Playwright harness opens a fresh browser session per query to keep the results stable.

Known Issues And Follow-Up

  • The generated-code safety gate is heuristic. It blocks imports, file I/O, network access, and obvious unsafe constructs, but it is not a fully sandboxed Python runtime.
  • LLM output is inherently probabilistic. The prompt and retry loop produced stable acceptance results here, but future model changes can still affect wording or formatting.
  • The current build focuses on single-workbook conversational analysis. Multi-sheet selection, result export, and downloadable chart artifacts would be sensible follow-up work.
