  • West region profit of ~$110,798.82 is acceptable as "110,799 (rounded)" per acceptance criteria
  • API key persistent storage uses a JSON file at ~/.talk2excel/config.json (no encryption, but file-system permissions apply)
  • LLM-generated code is executed via Python exec() in the Streamlit server process (acceptable for local-only use)

Known Issues & Limitations

  • Security: LLM-generated code is executed via exec() without sandboxing. Acceptable for local use but not for multi-user deployment.
  • Non-deterministic LLM responses: The LLM may occasionally generate syntactically invalid Python code. Error messages are displayed in the chat but no automatic retry is implemented.
  • API key storage: The key is stored as plaintext JSON on disk. For production use, a system keyring integration would be recommended.
  • Large files: Very large Excel files may cause slow processing or memory issues since the full DataFrame is held in memory.
  • Chart rendering: Charts are rendered server-side as static PNG images via matplotlib. Interactive charts (e.g., Plotly) are not supported.
